# `data_raw/` ŌĆö Raw Input Data: Acquisition Guide and Data Dictionary

All external input data for the ECM_manuscript pipeline. **This directory is
read-only**: the pipeline never writes here, and nothing in it is produced by
the project's own scripts. Everything the pipeline generates goes to
`data_derived/` instead.

Most files were obtained from external sources and copied into this directory.
A few small base layers (administrative boundaries, ecoregions, Natural Earth,
FungalTraits) are re-downloaded automatically by `01_spatial_data.R` /
`04_combine_ecm_dataset.R` if absent, but copies are already present here.

> **Integrity note.** Two large files dominate this directory: the GlobalFungi
> SH abundance matrix (~13 GB) and the WorldClim climate raster (~1.3 GB). If a
> download is truncated the pipeline can fail silently, so expected sizes are
> given for each entry below.

distributed research resource, and we cite all four; if any later states terms
that conflict with redistribution, the affected derived files can be withdrawn.

**The most restrictive terms govern the archive.** Two consequences:

1. **The derived data are released under CC BY-NC 4.0.** FungalRoot and the
   GBIF occurrence download are NonCommercial, and derived products that
   incorporate them cannot carry a more permissive licence. CC0 is therefore
   not available for this deposit, despite being a common Dataverse default.
2. **No GADM- or WorldClim-derived file is deposited.** Both prohibit
   redistribution. `data_derived/spatial/canada_simple.gpkg` (GADM-derived) is
   excluded from the archive and regenerated by `01_spatial_data.R`; the
   climate-space summaries are aggregate statistics, not redistributions of the
   WorldClim rasters.

No source imposes a NoDerivatives or ShareAlike term, so no upstream licence
propagates onto the derived data beyond the NonCommercial condition above.

## GlobalFungi v5

**Location**: `data_raw/GlobalFungi/`
**Source**: https://globalfungi.com ŌåÆ Downloads
**Citation**: V─øtrovsk├Į T et al. (2020) GlobalFungi. *Scientific Data* 7: 228.
**Licence**: Not indicated by the source. GlobalFungi aggregates sequence data from the NCBI Sequence Read Archive and other public repositories; the underlying reads are publicly available, but GlobalFungi states no licence for the compiled database.

| File | Size | Description |
|---|---|---|
| `GlobalFungi_5_sample_metadata.txt` | ~78 MB | Tab-delimited sample metadata (one row per sample; country, coordinates, barcoding region, sample type, dominant/other plant species, etc.) |
| `GlobalFungi_5_SH_abundance_ITS1_ITS2.txt` | ~13 GB | Wide-format SH abundance matrix: rows = samples, columns = UNITE v10 SH codes, values = read counts |
| `Metadata_table_description.pdf` | ~80 KB | Field descriptions for the metadata table (from GlobalFungi); reproduced in the table below |

**Filters applied by pipeline** ŌĆö `02_globalfungi.R` retains only:

- `country == "Canada"`
- `barcoding region %in% c("ITS2", "ITSboth")`
- `manipulated == "NO"`
- sample substrate not in `c("shoot", "air", "water", "sediment")`

then extracts the matching columns/rows from the abundance matrix using `awk`
and pivots to long format, dropping zero-abundance records.

> The GlobalFungi species-level abundance matrix
> (`GlobalFungi_5_species_abundance_ITS1_ITS2.txt`, ~4 GB) is **not used** by
> this pipeline and is not included here ŌĆö the global Wallacean analysis works
> from the SH-level matrix decoded through our own pinned UNITE build.

## UNITE General FASTA (SH reference)

**Location**: `data_raw/UNITE/`
**Source**: https://unite.ut.ee/repository.php ŌåÆ "General FASTA releases"
**Citation**: Nilsson RH et al. (2019) *Nucleic Acids Research* 47: D259ŌĆōD264.
**Licence**: CC BY 4.0 ŌĆö https://unite.ut.ee/cite.php

| File | Size | Description |
|---|---|---|
| `sh_general_release_dynamic_04.04.2024_dev.fasta` | ~76 MB | **Pinned / required.** UNITE v10.0 general FASTA, build 2024-04-04, `_dev` (full-length) variant, DOI `10.15156/BIO/2959332`. One representative sequence per SH code (93,085 SH codes). |

**Format**: standard FASTA. Each header line encodes
`>accession|species|SH_code|taxonomy` (pipe-delimited); the sequence sourced
from the header's SH_code and taxonomy string is what the pipeline parses into
`unite_sh_taxonomy.csv` (a checkpoint).

**Why this build is pinned (not the newest available).** UNITE SH identifiers
are renumbered between dated builds of the same major version. GlobalFungi's
pre-assigned SH codes were generated against the 2024-04-04 build (which
resolves >99% of them, versus ~3% for the 2025-02-19 build), and GenBank SH
codes are assigned in-house by `vsearch` against this same file, so both
sources decode consistently. `paths$unite_fasta` in `00_setup.R` points every
role at this one file. Re-pin to a newer build only after confirming SH-code
match coverage is still high for the GlobalFungi component.

## Live-query sources (no file in `data_raw/`)

Five inputs are queried directly from web services rather than downloaded as
files, so they have no entry above. Each response is cached in
`data_derived/checkpoints/` on first run and reused thereafter, which is what
makes the pipeline re-runnable without re-querying.

**These services have no versioned releases.** A query re-run later will not
return the same result: taxonomies are revised, records are added, and native-
status checklists are refreshed. The cached responses are therefore the only
record of what the reported numbers were computed from, and are deposited in
the Borealis archive for that reason (see `ARCHIVING.md`).

| Source | Queried by | Purpose | Cached as | Licence |
|---|---|---|---|---|
| **NCBI GenBank** (Entrez) | `03_genbank.R`, `18_eltonian.R` | Canadian fungal ITS records and their metadata; global EcM records per genus for the Eltonian comparison | `genbank_emf_canada_metadata.csv`, `genbank_global_ecm_meta.csv`, and companion FASTA/vsearch files | No NCBI-imposed restrictions; submitters may retain rights |
| **GBIF Backbone Taxonomy** | `05_prepare_fungalroot.R` (`rgbif::name_backbone_checklist()`) | Resolves plant name synonymy in FungalRoot to accepted names | `fungalroot_species_backbone.csv` | CC BY 4.0 |
| **BIEN** | `06_host_species.R` (`BIEN::BIEN_list_country()`, `BIEN_trait_traitbyspecies()`) | Vascular plant species recorded for Canada; growth-form traits | `bien_nsr_native_species.csv`, `bien_ecm_growthforms.csv` | CC BY 4.0 |
| **NSR** (Native Species Resolver) | `06_host_species.R` (`NSR::NSR_simple()`, batched 500 spp/request) | Native vs introduced status for each Canadian species | `bien_nsr_native_species.csv` | CC BY 4.0 |
| **GIFT** | `06_host_species.R` (`GIFT::GIFT_traits()`, trait 1.2.1, `agreement = 0.66`) | Growth forms for species BIEN does not cover (15 of 272 species) | `gift_growthforms.csv` | Not indicated by the source |

### NSR's underlying checklists

NSR does not hold native-status data of its own; it compares each species
against curated regional checklists. Two of those govern the Canadian
determinations that produce our host list, and both are separately licensed:

| Checklist | Role | Refreshed by NSR | Licence |
|---|---|---|---|
| Database of Vascular Plants of Canada (VASCAN) | The Canadian native-status authority | 16 September 2024 | CC0 1.0 ŌĆö GBIF dataset DOI 10.5886/zw3aqw |
| World Checklist of Vascular Plants (WCVP) v13, delivered via POWO | Global names and distribution backbone | 21 May 2024 | CC BY 4.0 ŌĆö GBIF dataset `f382f0ce-323a-4091-bb9f-add557f3a9a2` |

NSR draws on twelve checklists in total (<https://nsr.biendata.org/sources/>);
the other ten cover regions outside Canada and do not affect this analysis.

**Access dates.** GenBank was queried 29 June 2026. The NSR checklist refresh
dates above are properties of the service, not of our query.

**API keys.** GenBank access benefits from an `ENTREZ_KEY` (see the root
`README.md`). The BIEN, NSR, GIFT and GBIF-backbone queries need internet
access but no credentials.
